# Layout & UI Refactor Guide

## 目标
我们需要对当前系统的前端布局和部分页面的 UI 进行一次彻底的重构和升级（主要参考草图 `web/public/mockups/lingpai-guanli-v3.html`），以达成全新的视觉体验，**但必须保证原有的所有业务逻辑、API 请求、组件状态和渲染条件完好无损。**

## 核心任务拆解

### 1. 全局布局重构 (Layout)
当前系统的全局布局（`HeaderBar`、`SiderBar` 以及主页面容器 `Layout` 等）需要更换为草图中的无边框、扁平化、全屏铺满的沉浸式布局。

**你需要做的是：**
- **重写 HeaderBar**：
  - 弃用当前 HeaderBar 中的旧版边框和背景，改用草图中的全透明+毛玻璃效果（`backdrop-filter: blur(12px)`）。
  - 左侧 Logo 区域精简，直接显示图标和文字，去除非必要的边框。
  - 居中部分实现导航菜单（首页、控制台、模型广场、文档）的顶部平铺。
  - 右侧功能区（头像、聊天、通知、暗色模式切换）保持当前的功能逻辑，但替换为草图中的悬浮圆形图标和无边框下拉。
- **重写 SiderBar**：
  - 将 SiderBar 的宽度缩小至 170px 左右。
  - 移除原来的纯色块状背景，改用透明+边框分隔的轻量化样式。
  - 菜单项 (Menu Item) 的激活态（active）样式改为淡紫色底 + 紫色图标和文字，非激活态仅为灰色文字。
  - 字体大小和内外边距严格对照草图进行调整，确保上下紧凑（字体15px，分类标题13px）。
- **更新 Layout Wrapper**：
  - 去除 `body` 或 `Layout` 层面写死的四周 `padding`（如旧版的 `padding: 30px`）和最大宽度限制（如 `max-width: 1560px`），使得页面可以 `100vw` 满宽铺展。

### 2. "令牌管理" (Token) 页面 UI 重构
令牌管理页面的视觉呈现需要彻底改头换面，从传统的“表格视图”转变为草图中的“现代卡片列表视图”。

**你需要做的是：**
- **重构列表页 (Token/index.jsx)**：
  - 顶部操作栏（添加令牌、搜索框、操作按钮）取消大面积阴影和死板边框，改为轻量级的淡色块与圆角输入框。
  - 放弃原生的、带有生硬边框的 `Table` 组件渲染方式。
  - 把每一行数据重写为一个卡片式横条（参考草图的 flex 布局），包含：复选框、名称、状态、额度、分组、密钥预览，以及右侧固定的操作按钮组（禁用、编辑、删除）。
- **保留核心逻辑 (最重要)**：
  - **绝不能修改任何网络请求逻辑**：`loadTokens`、`searchTokens`、`addToken`、`deleteToken` 等方法必须原样保留。
  - **绝不能修改状态管理**：`useState` 里的 `tokens`、`loading`、`searchKeyword` 等变量定义不可更改。
  - **绝不能修改权限控制**：管理员/普通用户的区分逻辑、按钮的鉴权展示必须保持原样。
